# 売場分析

このメニューは、BiZOOPeのオプション機能です。

利用する為には「棚割データ」の連携が必要です。

このメニューでは価格帯／プライスポイントを軸にした、商品構成グラフ、プライスポイント検証、価格帯検証、単品情報の4つの帳票から品揃え、フェイシングを検討する事ができます。

ID-POSならではの指標”ID数”にも着目してみて下さい。

BiZOOPeの他のメニューと異なり、出力は分析画面では無くPDF、EXCEL、CSVのファイルダウンロードによって行われます。

1．商品構成グラフ（PDFより抜粋）

2．プライスポイント検証（PDFより抜粋）

3．価格帯検証（PDFより抜粋）

4．単品情報（PDFより抜粋）

# 棚割パターン指定

インターフェイスされた棚割データの中から分析に使用する棚割パターンを、パターンコード／パターン名称のあいまい検索から指定する事ができます。

棚割パターンを指定する事で、そこに含まれる店舗、商品の指定を行った事となります。

「棚割パターンを検索する」リンクをクリックする事で、以下のダイアログがポップアップされます。

「検索ワード」欄にパターンコードまたはパターン名称の一部を入力する事で、該当する候補がリストとして表示されます。

分類を指定する事で、あいまい検索の対象とする棚割パターンを絞り込む事が出来ます。

何も指定しない場合、全ての棚割パターンの中からあいまい検索を行います。

リストには集計期間内の棚割データから、最新の週開始日※を持つ棚割パターンがパターンコード昇順に並びます。

多くの場合、一つの棚割パターンを選択して分析を行いますが、パターンの管理方法は企業／部門によってまちまちですので、ケースに合わせて棚割パターンの選択を行って下さい。

「選択分類外の商品を表示・集計から除外」にチェック入れる事で、指定した分類以外の商品が棚割パターン中に含まれていた場合でも、分析時にはこれを棚割パターンから除外して集計・表示を行います。

図の例で言えば、棚割パターン「カレー・シチュールー」の中から分類＝「カレー・シチュー即席」に該当する商品だけが集計・表示され、分類＝「香辛料」や分類＝「カレー・シチューレトルト」に該当する商品は、たとえ当該棚割パターン中に陳列されていたとしても、集計・表示から除外されます。

※.週開始日＝当該棚割パターンの売場における適用日。尚、棚割データは実際の更新有無に関わらず週単位で全件を連携頂く仕様となっており、BiZOOPeでは週開始日が当週含めて79週間内の棚割データを保持します。

# 商品構成グラフ設定

商品構成グラフに関わる設定を行います。

商品構成グラフは縦軸1（左）に”仮説”である棚割の指標を取り、縦軸2（右）に”検証”である顧客の利用実績の指標を取る事で対比し、横軸に配された「価格帯」毎に考察を行えるようにしたグラフです。

# 縦軸1（棚割）

フェイス数またはSKU数から選択できます。

# 縦軸2（実績）

点数、金額、ID数から選択できます。

# 横軸（価格帯）

価格帯を幾ら刻みで作成するかの入力を行います。

「最小グラフ単価から表示する」のチェックを外した場合、棚割データ中に存在する最低価格帯からグラフの作成を行います。

デフォルト（チェックonの状態）では、棚割データ中にその価格帯の商品が存在していなくても、必ず刻みの最小値からグラフの作成を行います。

# 年代設定

集計値「選択合計」とグラフ表示に関する設定です。

ここでの選択の影響は実績値に限定され、棚割データ側に影響を与える事はありません。

# 集計

各帳票の「選択合計」として集計する年代を選択します。

「非会員」の選択／非選択は点数、金額及びそれらに関連する指標に影響を与えますが、ID数には影響を与えません。

# 表示｜年代グラフ

年代別の商品構成グラフ上に表示したい年代を選択します。

前出「集計」において選択されていない年代を選択する事はできません。

# 1．商品構成グラフ

棚割と「選択合計」とを比較する商品構成グラフ（左側）と、各年代実績とを比較する年代別商品構成グラフ（右側）の2つのグラフが出力されます。

左側のグラフの「選択合計」はオプション＞年代設定の「集計」欄における指定を、右側のグラフの各年代実績は「表示｜年代グラフ」欄における指定を反映したもので、右側の年代別グラフには「選択合計」を含みません（凡例は両グラフ共通のものとして左右両端に集約して表現されています）。

いづれのグラフも縦軸左側は棚割（図ではフェイス数）、横軸は価格帯で共通、同一スケールで引かれています。

いづれのグラフも縦軸右側は実績（図ではID数）を表し、各グラフ毎の実績の最大値を上限値としたスケールがそれぞれで引かれています。

これにより、棚割については左右のグラフで全くの同形で表現されており、それに対する各々の実績の比較もしやすい形で表現されています。

例えば図の左のグラフからは、利用者の数に比較し「〜200円迄」のフェイスが広く、「〜250円迄」のフェイスが狭い事が仮説されます。

また、前者のフェイスを減らすとすれば／後者のフェイスを増やすとすれば、どの年代向けのものを対象とするか？を右の年代別グラフから検討する事ができます。

# 2．プライスポイント検証

表は大きくメインである「棚割り単価」を表示する行グループ、実勢である「点単価」を表示する行グループ、それらの根拠となる「設定値／実績値」を表示する行グループの3つに分かれています。

また、各列の項目の値が最大値を示す「第一ポイント」、それに次ぐ値を示す「第二ポイント」の表示が本表の特徴となります。

各列の項目の値とは無関係に、指定した棚割パターン中に含まれる棚割単価の最低値、最高値が「最低ポイント」、「最高ポイント」です。

表中の各値の詳細については後述します。

本表の特記事項について以下に記します。

# 各ポイント棚割り単価

主に棚割り単価が表示されています。

・ポイント数

プライスポイントの数、すなわち単価のバラエティーを表します。

企業によっては売場の価格構成をシンプルに、顧客に分り易くする為にポイント数を一定数以内に抑えるようにしています。

・第一／第二

各列の項目の値が最大値を示す棚割り単価が「第一ポイント」、それに次ぐ値を示す棚割り単価が「第二ポイント」です。

図ではフェイス数、SKU数、金額の第一ポイントが258円、点数、ID数の第一ポイントが218円となっていますが、利用者数が多いほど嗜好の多様性も増えるものと仮説すれば、フェイス数、SKU数を258円から218円に寄せるといった事が考えられます。

・最低／最高と倍率

企業によってはストアコンセプトを逸脱しないように 最高ポイント ÷ 最低ポイント の倍率を一定値以内に保つようにしています。最高／最低双方の単価と併せて検討が行われます。

# 各ポイント点単価実績

主に点単価（金額 ÷　点数）が表示されています。

特に第一ポイント、第二ポイントにおいて点単価実績を見る事で、実績がほぼ棚割り単価付近で記録されたものなのか、よりマークダウンに依存したものなのか（値決めは本当に適当か）を検討します。

図では点数、ID数の第一ポイントにおける点単価が、棚割り単価の218円を大きく下回った値を記録している事が分かりますので、実際にフェイス数、SKU数を寄せるべき単価とは幾らなのか？の検討材料となります。

# 各ポイント設定値／実績値

各ポイントで記録された実際の設定値、実績値が表示されます。

図では第一ポイントと第二ポイントのフェイス数が微差（1差）であるのに対して、SKU数の差が大きい（5差）事が分かりますので、フェイス割当が適当で無い事が疑われます。

# 3．価格帯検証

商品構成グラフの内訳を表す帳票です。

EXCELの方でフェイスあたり売上やID数、SKUあたり売上やID数を計算してみると商品構成グラフとはまた違った実態が見えてくるかもしれません。

# 4．単品情報

全ての帳票の内訳となる帳票です。

主に単品の棚割り単価がどのように採用され、グラフ単価が計算、決定されているかを示しています。

また、PI（1,000人あたり）は企業によっては商品カットの基準値として用いられています。

# 商品毎来店客数について

PI（1,000人あたり）算出の分母となる来店客数です。

棚割りデータを用いる事でPI（1,000人あたり）をより正確な（？）値とする為、指定された棚割パターン内で商品毎に設定されている取扱店舗、陳列期間に基づいてジャーナルデータからレシート枚数を算出しています。

PI（1,000人あたり）の分母は商品の利用客数では無く、あくまでも”来店客数”である事から、「商品」はその取扱店舗、陳列期間の抽出のみに利用され、レシート抽出にあたっての検索条件には含まれません。

・店舗

指定された棚割パターン内で商品毎に設定された取扱店舗を商品毎来店客数の集計対象としています。

・期間

指定された棚割パターン内の店舗-商品の組み合わせ毎に設定された陳列期間内、且つ指定された集計期間内である事を条件に商品毎来店客数の集計対象としています。
